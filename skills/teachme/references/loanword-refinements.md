# 외래어 순화 지침 — 한국어 검수(§7) 서브에이전트용

> 이 문서는 외래어를 만났을 때 **서브에이전트가 따를 판단 지침**이다.
> **고정된 치환 사전(DB)이 아니다** — *언제·어떻게 바꿀지*의 기준과 대표 예시를 준다.
> 애매한 경우는 기계적 조회가 아니라 **아래 원칙으로 문맥에 맞게 판단**한다(정확한 지침 + 판단 > 사전 일괄 적용).
>
> **근거 자료(권위 출처):** 국립국어원 **「다듬은 말」**
> <https://www.korean.go.kr/front/imprv/refineList.do> · 우리말 다듬기 <https://malteo.korean.go.kr> (약 2.1만 건).
> 판단이 서지 않거나 처음 보는 외래어는 여기서 **근거를 확인**한 뒤 판단한다(추측 금지). 단, 사전에 다듬은 말이 있다고 해서 무조건 쓰지는 않는다(§3 참고).

---

## 0. 핵심 원칙 — 사전을 "근거"로, 적용은 "판단"으로

1. **근거는 사전이다.** 어떤 외래어를 어떤 한국어로 바꿀지는 국립국어원 「다듬은 말」을 근거로 삼는다. 예시 몇 개를 외워 쓰거나 임의로 지어내지 않는다.
2. **그러나 사전을 기계적으로 적용하지 않는다.** 다듬은 말 중에는 실무에서 정착되지 않아 오히려 어색하거나 뜻이 흐려지는 것도 많다(예: 인터넷→누리망, 로그인→접속). **대상 독자에게 더 자연스럽고 명확할 때만** 바꾼다.
3. **정착된 기술 식별자·코드는 유지한다.** `API`, `HTML`, `CSS`, `URL`, `SVG`, `JSON`, `pipeline()`, `Workflow`, `git` 등은 그대로 둔다.
4. **비유(calque)는 사전 항목이 아니다.** "렌즈"(검토 렌즈), "드리프트", "게이트(비유)" 같은 표현은 외래어 순화 대상이 아니라 **영어 비유의 직역**이다 → 문맥에 맞는 한국어로(렌즈→관점, 드리프트→어긋남). 이는 사전이 아니라 §7의 "직역투" 관점으로 판단한다.

---

## 1. 바꾼다 — 정착됐고 더 자연스러운 순화어 (국립국어원 다듬은 말)

| 외래어 | 다듬은 말 |
|--------|----------|
| 가이드라인 | 지침 |
| 다운로드 | 내려받기 |
| 업로드 | 올리기 |
| 업데이트 | 갱신 |
| 모니터링 | 점검 / 감시 |
| 게이트웨이 · 게이트 | 관문 |
| 프로토콜 | 통신 규약 |
| 로그 · 로깅 | 기록 |
| 트래픽 | 소통량 |
| 스토리지 | 저장 장치 |
| 솔루션 | 해결책 |

## 2. 맥락 판단 — 과하면 원어 유지

| 외래어 | 다듬은 말 | 판단 기준 |
|--------|----------|-----------|
| 콘텐츠 | 내용(물) | "콘텐츠"가 널리 정착 — 문맥상 "내용"이 자연스러우면 바꾸고, 아니면 유지 |
| 플랫폼 | 기반 | 일반 문맥은 "기반", 고유 제품/기술 맥락은 유지 |
| 인프라 | 기반 시설 | 길어지면 원어가 더 명확할 수 있음 |
| 프로세스 | 절차 · 과정 | 대부분 "절차/과정"이 자연스러움 |

## 3. 쓰지 않는다 — 사전엔 있으나 비정착 (원어 유지)

아래는 국립국어원 다듬은 말이 존재하지만 **실무에서 통용되지 않아 그대로 두는** 예다. 사전을 기계적으로 적용하면 안 되는 이유의 사례이기도 하다.

- 인터넷 → ~~누리망~~ · 로그인/로그아웃 → ~~접속/접속 해지~~ · 웹서핑 → ~~누리 검색~~
- 와이파이 → ~~근거리 무선망~~ · 아이콘 → ~~그림 단추~~ · 메신저 → ~~쪽지창~~

> 이 항목들은 "다듬은 말이 있다 ≠ 반드시 쓴다"를 보여 준다. §0-2 원칙을 따른다.
